# Cloud Security Monitoring & Threat Detection Platform on AWS

## About the Project

I built this project to gain hands-on experience with cloud security monitoring, threat detection, and security operations in AWS.

The goal was to create a small-scale Security Operations Center (SOC) environment that could monitor cloud infrastructure, web applications, and network activity from a central location. Instead of working with isolated tools, I wanted to understand how security teams collect logs, investigate incidents, and respond to threats in real-world environments.

The environment consists of two web servers hosting DVWA behind an AWS Application Load Balancer, a dedicated Splunk SIEM server, AWS CloudTrail for audit logging, VPC Flow Logs for network visibility, AWS WAF for web application protection, and an AI-powered security assistant for alert analysis.

---

## What I Built

* Deployed a multi-server environment in AWS using EC2.
* Configured an Application Load Balancer to distribute traffic across two web servers.
* Installed DVWA to simulate real-world web application attacks.
* Set up Splunk Enterprise as a centralized SIEM platform.
* Configured Splunk Universal Forwarders to collect logs from web servers.
* Enabled CloudTrail to monitor AWS account activity.
* Enabled VPC Flow Logs to capture network traffic metadata.
* Configured AWS WAF to protect the application from common web attacks.
* Created security dashboards and detection rules in Splunk.
* Designed an AI security agent concept to help analyze and prioritize security alerts.

## Why This Project Matters

Cloud environments generate a large amount of security data every day. Without centralized monitoring, it becomes difficult to identify suspicious activity or investigate incidents.

This project helped me understand:

* How cloud logs are generated and collected.
* How SIEM platforms ingest and analyze security events.
* How threat detection rules are created.
* How attackers interact with vulnerable applications.
* How security analysts investigate suspicious activity.
* How cloud security services work together to improve visibility.

## Security Monitoring

The platform collects and analyzes data from multiple sources:

### AWS CloudTrail

Used to monitor:

* AWS API activity
* IAM changes
* Infrastructure modifications
* Administrative actions

### VPC Flow Logs

Used to monitor:

* Network traffic
* Source and destination IP addresses
* Allowed and denied connections
* Suspicious communication patterns

### Web Server Logs

Used to monitor:

* HTTP requests
* Application errors
* User activity
* Attack attempts

### Authentication Logs

Used to monitor:

* Failed SSH logins
* Successful logins
* Brute-force attempts

---

## Attack Simulations

To test the monitoring environment, I performed several attack simulations against DVWA:

* SQL Injection
* Cross-Site Scripting (XSS)
* Brute Force Authentication
* Directory Traversal
* Nmap Reconnaissance Scanning

The generated logs were collected by Splunk and used to validate detection rules and dashboards.

## Detection Rules

Some of the detections implemented in Splunk include:

* SSH Brute Force Detection
* SQL Injection Detection
* XSS Detection
* Port Scan Detection
* Suspicious Login Detection
* Unauthorized AWS Activity Monitoring

These detections help identify potentially malicious behavior and improve investigation speed.

## AI Security Agent

As an extension of the project, I designed an AI Security Agent concept that consumes alerts generated by Splunk and assists with investigation.

The idea is to automatically:

* Summarize alerts
* Assign severity levels
* Explain why an event is suspicious
* Recommend next investigation steps
* Generate incident summaries

This simulates how AI is increasingly being used in modern SOC environments.
